接口隔离原则

简单来说，就是一个接口中声明的方法，不应该是对子类没有用的方法。更广泛的说，父类接口中声明的方法，应该是对子类有用的，对子类没有用的方法就不要出现在父类接口中。

那么什么是“没有用的方法”呢？其实就是指父类接口中声明了的方法，但是在它众多子类中，有些子类实现了其中一些方法的逻辑代码，有些子类实现了另一些方法的逻辑代码。

所以，为了防止这种复杂的情况出现，应当对父类接口中的方法进行一个分类，将其中一些方法移动到新的接口下。

总结：一个实现类中，根据这个类要实现的功能去继承所需要的方法。

例如：我有一个接口，并且它有两个子类。

interface AutoMobile { void repair(); void run(); } class AutoMobileImpl implement AutoMobile{ void repair(){ } void run(){}//这个方法在当前类的设计中并没有使用到。 }
