依赖倒转原则

当我们设计好一系列的逻辑代码后，如果需要增加功能，就需要在对应的地方增加相应的逻辑代码。

如：


    class Email {
        public String getInfo() {
            return "电子邮件信息";
        }
    }
    
    class Person {
        public void receive(Email email) {
            System.out.println(email.getInfo());
        }
    }
    
    public static void main(String[] args) {
        Person person = new Person();
        person.receive(new Email());
    }

如以上代码，当我们获取信息的对象是微信，短信等等，则需要增加类，同时Person也要增加相应的接收方法。我们可以利用多态性质使代码变得简单一些。 

        public interface IReceiver {
            public String getInfo();
        }
        
        class Email implements IReceiver {
        
            @Override
            public String getInfo() {
                return "电子邮件接收信息";
            }
        }
        
        class WeiXin implements IReceiver {
        
            @Override
            public String getInfo() {
                return "微信接收信息";
            }
        }
        
        class Person {
            public void receiver(IReceiver receiver) {
                System.out.println(receiver.getInfo());
            }
        }
        
        public static void main(String[] args) {
            Person person = new Person();
            person.receiver(new Email());
            person.receiver(new WeiXin());
        }

使用继承或者实现性质的情况：当我们需要实现的功能与原有功能相似的时候，就可以使用两性质。
